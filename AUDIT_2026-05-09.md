# Hermes Affiliation Machine — Audit & Resolution Log

> **Audit date:** 2026-05-09
> **Auditor:** Cursor / Claude Opus 4.7
> **Scope:** event-driven Hermes pipeline, 5 production sites, 8-agent roster, GSC + Apify integrations

---

## 0. Executive summary

The Hermes architecture is solid (10 agents, file-based event bus, GSC + Apify + Resend wiring, IndexNow infra, schema coverage), but it accumulated a layer of silent integration bugs that quietly capped revenue and broke the analytics signal:

- **2 entire ad-tech stacks** (Meta Pixel + GA4 EU regional collectors) were CSP-blocked on 4 of 5 sites — confirmed in Playwright debug reports.
- **2 production sites** were never CRO-tested (wrong domains in `cro-link-tester.ts` since day one).
- **Every refresh brief** told the writer to publish "...2025" headlines, in 2026.
- **Newsletter HTML** rendered as raw `**markdown**` for every bold pair after the first.
- **Revenue tracking** is fake (`avg_order_value=150` hardcoded for matelas-vs-coffee), so the strategist has been flying blind.
- **IndexNow** routes existed on every site but no agent ever called them (so new content waited days for indexing).

Plus a P0 security gap: live API keys (Anthropic, Gemini, Resend, Apify) and a Google service-account JSON were sitting outside any `.gitignore`.

This document tracks every finding and its resolution.

Publisher consumes the event, performs `git revert <bad-sha>`, opens a **manual-review PR** (no auto-merge on rollbacks), and emits `deployment.rollback_proposed`. |
